# operantlunar: Differentiating Reward Maximization from Melioration via Operant Conditioning

A tabular reinforcement-learning instrument that mechanistically
differentiates reward-maximizing control (the law of effect) from
descriptive melioration (the matching law). Provides a shared learning
skeleton; a library of learning rules (Q-learning, SARSA, expected
SARSA, double Q-learning, actor-critic, a model-based planner,
gradient-bandit and rate-tracking melioration, win-stay-lose-shift); a
battery of operant paradigms (schedule family, melioration trap,
extinction, probability matching, self-control / delay discounting,
differential reinforcement of low rates, progressive ratio, risk
sensitivity, reinforcer devaluation, changeover-delay concurrent
schedules); an analysis layer with generalized-matching,
Herrnstein-hyperbola and discounting fits and a differentiation-matrix
capstone; hashed tile coding with linear function-approximation agents
and a generic Gymnasium adapter; an optional Python LunarLander
environment via reticulate; and a bundled Shiny app.
